Overview

This file is part of the converter Add-on that allows loading and saving of CANdela documents (*.cdd and *.cddt) created by different versions of CANdelaStudio. The converters are used by CANdelaStudio and by Vector tools that support loading CANdela documents (e.g. CANdelaStudio, CANdelaFlash, CANdito, CANape Graph, CANoe, CANdesc).

The converters contained in this Add-on support CANdela documents up to CANdelaStudio 7.1 and Service Packs (CANdelaStudio 7.1 was released 2011-07-27).

2.0 -> 3.0

  • set mandatory at runtime = false only for 0x2a 0x04 (not for other 0x2a ProtocolServices). EIP9164 "Adapt mandatoryAtRuntime migration 2.0=>3.0"

See also EIP9564 "Persistor in CANdelaStudio 5.5 Release fails to load 2.0 files".

After CANdelaStudio 5.0 SP 2 (2007-02-12)

CANdelaStudio 3.0 together with the converter shipped with CANdelaStudio 5.0 Release is not able to load documents created with CANdelaStudio Service Packs after 5.0 Release (e.g. with 5.0 SP 2).

This updated converter allows CANdelaStudio 3.0 (and all clients based on CANdelaStudio <= 4.0) to load such documents (i.e. documents created with all CANdelaStudio 5.0 Service Packs). Note that this was already possible for clients based on CANdelaStudio >= 4.0.

With CANdelaStudio 5.0 Release (2006-08-02)

Support new data model of CANdelaStudio 5.0.

With CANdelaStudio 4.0 SP 4 (2006-05-23)

All versions

  • Tolerate UTF-8 files containing Byte Order Mark (e.g. from Notepad) EIP5487 "Add new expat to 1_3 and 3_4 Persistors"

2.0 -> 3.0

  • Avoid duplicate qualifiers EIP5886 "Correct migration of SharedProxyComp qualifiers from 2.0 -> 3.0"
  • Do not reject migration of files containing special EcuDoc qualifier EIP5437 "CDD with dtdvers='1.4.5' not loadable in CANdelaStudio 3.0"

3.0 -> 4.0

  • Allow migration of files created with CANdelaStudio 3.0 Release (not SP) to 4.0 EIP5180 "Files saved with released CANdelaStudio 3.0.4 are not loadable in CANdelaStudio 4.0"
  • Repair inconistent communication parameters EIP5213 "Migrate communication parameter properties to be consistent"

4.0 -> 3.0

  • Avoid loss of byte order for data objects in Packet Datatype (EIP5725)

3.0 -> 2.0

  • Avoid issue with duplicate data objects EIP4783 "It is not possible to save the .cdd file (see attachment on 2.0.5 format")
